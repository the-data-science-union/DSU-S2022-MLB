Here are the things I took away from running the .info() method:

  • The 16th column, 1stIP, has 0 non-null values -- this probably means I should drop that column
  • SV% and IS% are listed as type object (type object means string or mixed data type). From the table on Baseball Reference, I expected these to be numbers, not strings -- this means I should edit these columns

Matplotlib is another popular plotting library in Python. There are two ways to plot in matplotlib:

Data Visualization with Matplotlib: Object-Oriented Example¶

In object-oriented matplotlib, the plt.subplots() function is used to create a grid of plots. plt.subplots() outputs a tuple containing the figure, which represents the grid as a whole, and the axes, which represent each individual plot in the grid.

Here are some examples:

fig, (ax1, ax2) = plt.subplots(2) [create 2 plots (by default it will make a 2x1 grid). Each plot will be accessed by ax1 and ax2]

fig, ax = plt.subplots(nrows = 2, ncols = 2) [create 4 plots on a 2x2 grid. Each plot will be accessed by ax[0] through ax[3]]

For those who are not familiar, GitHub is a website where data scientists can display their work by uploading their code, visualizations, and findings so that people can see your work.

There is a way to connect our project GitHub repository (folders/drives on GitHub are called repositories) to a folder on your computer so that when you make changes to code files on your computer, you can sync it to the GitHub repository (and vice versa sync the repository to your computer)! This is what we will be setting up today.

In order to sync folders on your computer to GitHub, you will need to open the shell (terminal) on your computer and type in Git commands

Using Git¶

Git is already installed in mac and linux systems, but if you have a windows system you will need to install the latest version of Git from the internet. Here is the link to Git for Windows.

When you first install Git, you will have to configure it. Make sure that the email in your configured Git is the same one used for your GitHub account. To configure Git, execute the following commands in your shell (terminal):

  • git config --global user.name "[insert GitHub username here]"
  • git config --global user.email "[insert GitHub email here]"

To check whether you have Git already installed, type git version into the shell. If it returns the version, you have it. If it doesn't, you will need to install it.

In addition, you will need to create a GitHub account on github.com if you haven't already.

The following slides will be a tutorial on how to set up your GitHub account and link out project's GitHub to your computer

Setting Up Our Project's GitHub Repository to Local Files on Your Computer¶

First, you will have to set up a SSH key for your computer on your GitHub account if you do not already have one. This allows GitHub to trust that the changes being made to the online repository from your computer are safe.

  1. On github.com navigate to your profile picture > Settings > SSH and GPG Keys
  2. In your shell, type ssh-keygen -t ed25519 -C "youremail@web.com" to generate a SSH Key, using the email address associated with your GitHub account. If you are the only person that uses your computer, I would reccommend not including a passcode. To do this, leave "enter passphrase" blank by hitting enter twice.
  3. Next, you will need to navigate to the .ssh folder on your computer using the shell. To do this, type cd .ssh in your shell (this changes the working direcotry to the .ssh folder). If you then type ls into the shell, it will list off the items in your current working directory. You should see an item that is named id_ed25519.pub.
  4. Next, type cat id_ed25519.pub (cat means print like cat() in R) to print out your SSH key. Copy the SSH key.
  5. Go back onto github.com, hit New SSH Key, and paste the SSH key into the Key box. Title the SSH key to the name of your computer and add it to your account.

Setting Up Our Project's GitHub Repository to Local Files on Your Computer¶

The next step is to clone (add) the repository to the folder you want it in on your computer. To do this, use cd in the shell to navigate to the directory (folder) on your computer you would like to place the repository in. I would suggest having your file system open and using tab completion so that you can navigate to the correct folder and ls to get a reminder of what is in the current folder.

Once your directory is set to the one you want the repository in (use pwd in the shell to check your present (current) working directory):

  1. Go to the repository on GitHUb and select Code > SSH > copy the SSH GitHub repository name
  2. In the shell, type git clone [SSH GitHub repository name]

The online reposiotry and the folder on your computer are now synced!

Important Git Commands for Syncing the Two Repositories¶

Before uploading and dowloanding files to and from the GitHub repository, make sure your current directory is the repository folder on your computer, or else you will encounter errors.

  • git status: tells you the current status of your repository. If you are new to Git, it really helps to type git status after every other git command, it will help to understand exactly what is happening.
  • git pull: takes the current version of the repository on GitHub and syncs it to your local repository (the folder of the repository on your computer). You should always use git pull before adding anything new to the repository so there are never any potential sync conflicts after other people add files.
  • git add [filename.file OR . (all files)]: adds new files/changes made to the staging area (deems those files ready to be committed (added) to the repository)
  • git commit -m "description": commits (makes the changes) to the repository and adds description of the changes that were made (i.e. "added the code file for my matplotlib graphs")
  • git push: takes the changes that were made on your local reposiotry and syncs them to the GitHub repository online so that everyone can see
